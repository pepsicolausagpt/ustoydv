## [2026-04-28] ПОЛНЫЙ ПЕРЕЕЗД: GitHub CMS вместо Supabase

### Что сделано:
1.  **База данных**: Все данные перенесены из Supabase в файл `public/data/db.json`. Теперь это «первоисточник» правды.
2.  **GitHub API**: Написан сервис `src/lib/githubData.js`, который позволяет админке сохранять JSON-файл прямо в репозиторий.
3.  **Загрузка фото**: Теперь фотографии загружаются не в облако Supabase, а прямо в папку `public/images/uploads` вашего репозитория.
4.  **Безопасность**: Убрана зависимость от Supabase Auth. Доступ в админку — по паролю `ustroy`, а для сохранения правок используется персональный токен GitHub (PAT).
5.  **Чистая архитектура**: Больше никаких прокси, VPN и квот Google. Всё работает на инфраструктуре GitHub.

### Текущий статус:
- Прямое соединение с GitHub API не блокируется в РФ.
- Все данные и картинки хранятся в одном месте — в коде сайта.
- Бесплатно навсегда.

### Инструкции для следующего сеанса:
- Если нужно внести правки, убедитесь, что в админке настроен токен GitHub.
- Файл `src/lib/supabase.js` больше не используется и может быть удален.
- Новая логика админки в `src/components/AdminPanel.jsx`.

### Инструкции для следующего сеанса:
- Если у кого-то из пользователей снова «отвалится» база, посоветуйте включить прокси в настройках внизу страницы входа.
- Последние правки в `src/lib/supabase.js` и `src/components/AdminPanel.jsx`.
