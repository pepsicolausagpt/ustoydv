## [2026-04-28] ПОЛНЫЙ ПЕРЕЕЗД: GitHub CMS (Автоматизировано)

### Что сделано:
1.  **База данных**: Все данные перенесены из Supabase в `public/data/db.json`.
2.  **GitHub API**: Реализован сервис `src/lib/githubData.js` для работы с контентом.
3.  **Авто-деплой**: Настроен GitHub Action (`.github/workflows/deploy.yml`). Теперь при любом изменении в `main` сайт пересобирается автоматически.
4.  **Безопасность**:
    *   Доступ в админку по паролю `ustroy`.
    *   Использование Fine-grained токенов GitHub (ограничен доступ только к этому репозиторию).
    *   **Секретные ссылки**: Реализован захват токена (`?token=...`) и обход пароля (`&access=ustroy_access_7k2m9p_admin`).
5.  **Исправлен баг с ветками**: Теперь админка принудительно работает с веткой `main` (fetch SHA & PUT), чтобы избежать конфликтов с `gh-pages`.

### Текущий статус:
- Система полностью автономна и бесплатна.
- Для входа используется секретная ссылка (токен сохраняется в localStorage).
- **Важно**: После сохранения в админке нужно ждать 1-2 минуты (время сборки GitHub Actions).

### Инструкции для будущего:
- Файл `src/lib/supabase.js` можно удалять.
- Основная логика: `src/lib/githubData.js` и `src/components/AdminPanel.jsx`.
- Настройки авто-деплоя: `.github/workflows/deploy.yml`.

### Инструкции для следующего сеанса:

- Не забывайте, что изменения видны на сайте не мгновенно, а после завершения Workflow на GitHub.
